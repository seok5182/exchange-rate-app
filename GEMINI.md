# 💱 프로젝트 컨텍스트 (Gemini Memory)

이 문서는 이 프로젝트의 아키텍처 설계 의도와 우리가 나눈 기술적 지식을 제가 잊지 않도록 기록한 '작업 기억' 저장소입니다.

## 📌 프로젝트 정체성
- **이름**: Exchange Rate App (환율 정보 조회기)
- **구조**: 관심사 분리(SoC)가 적용된 3계층 아키텍처 (`api` -> `services` -> `app.js`)
- **특징**: 대화형 CLI 인터페이스를 통한 실시간 환율 조회

## 🛠 핵심 설계 규칙 (전수된 철학)
1. **일관성 있는 데이터 정제**: 모든 입력 데이터(통화 코드 등)는 진입점(`app.js`)에서 대문자화(`toUpperCase`)하여 시스템 내부로 전달한다.
2. **Named Parameters 패턴**: 함수 인자는 객체`{}`로 묶어 전달하며, `{ key = value } = {}` 문법을 사용해 이중 안전장치를 구축한다.
3. **Config 중앙 관리**: `.env`의 값을 `src/config/appConfig.js`라는 '지갑'에 담아 타입 안정성과 유지보수성을 확보한다.
4. **따옴표 일관성**: 프로젝트 전체에서 작은따옴표(`'`)를 사용한다.
5. **확장자 명시**: ES Modules 환경이므로 `import` 시 반드시 `.js`를 붙인다.

## 💡 주요 디버깅 레슨
- **구조 분해 할당의 함정**: `const { rates } = data`와 같이 객체의 계층 구조를 정확히 파악하고 속성을 추출해야 한다. (잘못된 계층 참조 시 `undefined` 에러 발생)
- **보안 엔진의 우선순위**: 인터셉터는 브라우저의 보안 엔진(CORS 검사 등)보다 하위 계층이므로, 엔진이 파기한 데이터에는 접근할 수 없다.

## 🚀 향후 확장 계획

- 에러 핸들링 고도화 (존재하지 않는 통화 코드 예외 처리 등).
- API 호출 타임아웃 설정 추가.
